## Update Behavior

The `ksail cluster update` command applies configuration changes to a running cluster.
Changes are classified by impact: **in-place** (no disruption), **reboot-required** (node restart needed), or **recreate-required** (full cluster recreation).

| Change                   | Vanilla (Kind) | K3s (K3d) | Talos    |
| ------------------------ | -------------- | --------- | -------- |
| Distribution             | Recreate       | Recreate  | Recreate |
| Provider                 | Recreate       | Recreate  | Recreate |
| CNI                      | In-place       | In-place  | In-place |
| CSI                      | In-place       | In-place  | In-place |
| Metrics Server           | In-place       | In-place  | In-place |
| Load Balancer            | In-place       | In-place  | N/A¬π     |
| cert-manager             | In-place       | In-place  | In-place |
| Policy Engine            | In-place       | In-place  | In-place |
| GitOps Engine            | In-place       | In-place  | In-place |
| Local Registry           | Recreate       | In-place  | In-place |
| Mirrors Dir              | Recreate       | N/A       | N/A      |
| Control Planes           | Recreate       | Recreate  | In-place |
| Workers                  | Recreate       | In-place  | In-place |
| Hetzner Server Type (CP) | N/A            | N/A       | Recreate |
| Hetzner Location         | N/A            | N/A       | Recreate |
| Hetzner Network          | N/A            | N/A       | Recreate |

**Notes:**

- "In-place" changes are applied without cluster downtime via Helm or provider APIs
- "Recreate" changes require `ksail cluster delete && ksail cluster create` or `ksail cluster update --force`
- Use `ksail cluster update --dry-run` to preview changes before applying
- Talos supports the broadest set of in-place updates, including node scaling for both control-plane and worker nodes
- Kind does not support any structural node changes after creation
- **¬π Load Balancer for Talos+Docker**: LoadBalancer is currently not supported for Talos on Docker and is always `Disabled`. This row is N/A because the setting cannot be changed for this combination. See [FAQ](/faq/#which-distributions-support-loadbalancer-services) for more details.

## Version Compatibility

KSail embeds specific versions of Kubernetes tooling:

| Tool    | Embedded Version  | Purpose                    |
| ------- | ----------------- | -------------------------- |
| kubectl | Latest            | Kubernetes CLI             |
| Helm    | v4 (with kstatus) | Package manager            |
| Kind    | Latest            | Vanilla clusters           |
| K3d     | Latest            | K3s clusters               |
| Flux    | Latest            | GitOps toolkit             |
| ArgoCD  | Latest            | GitOps continuous delivery |
| SOPS    | Latest            | Secret encryption          |

**Notes:**

- Kubernetes versions depend on the distribution release
- Component versions (CNI, CSI, etc.) are updated with KSail releases
- See [releases](https://github.com/devantler-tech/ksail/releases) for specific version information
