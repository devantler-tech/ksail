[Kind (Kubernetes in Docker)](https://kind.sigs.k8s.io/) runs upstream Kubernetes clusters using Docker containers as nodes. It is fast (30–60 s), Docker-only, and uses standard `kind.yaml` files with no vendor lock-in. For common workflows and use cases, see [Use Cases](/use-cases/).

## Quick Start

Requires Docker ([installation prerequisites](/installation/)).

````bash
ksail cluster init --name my-cluster --distribution Vanilla --control-planes 1 --workers 2
ksail cluster create
ksail cluster info && kubectl get nodes  # verify
ksail cluster delete                     # cleanup
````

`ksail cluster init` creates `ksail.yaml` (KSail config) and `kind.yaml` (Kind cluster config).

## Configuration

### Basic Configuration

The `ksail.yaml` file controls your cluster:

````yaml
# yaml-language-server: $schema=https://raw.githubusercontent.com/devantler-tech/ksail/main/schemas/ksail-config.schema.json
apiVersion: ksail.io/v1alpha1
kind: Cluster
spec:
  cluster:
    distribution: Vanilla
    distributionConfig: kind.yaml
````

### Kind-Specific Configuration

Customize Kind behavior in `kind.yaml`:

````yaml
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
nodes:
  - role: control-plane
    # Port mapping for accessing services
    extraPortMappings:
      - containerPort: 80
        hostPort: 8080
        protocol: TCP
  - role: worker
  - role: worker

# Containerd configuration
containerdConfigPatches:
  - |-
    [plugins."io.containerd.grpc.v1.cri".registry.mirrors."localhost:5000"]
      endpoint = ["http://localhost:5000"]
````

### LoadBalancer Support

Enable LoadBalancer services with Cloud Provider KIND:

````yaml
# ksail.yaml
apiVersion: ksail.io/v1alpha1
kind: Cluster
spec:
  cluster:
    distribution: Vanilla
    loadBalancer: Enabled
````

When enabled, KSail installs Cloud Provider KIND, which creates `cpk-*` Docker containers as LoadBalancer proxies and assigns external IPs. See [LoadBalancer Configuration](/configuration/loadbalancer/) for details.

### Registry Mirrors

KSail enables `docker.io`, `ghcr.io`, `quay.io`, and `registry.k8s.io` mirrors by default. Override with `--mirror-registry` flags:

````bash
ksail cluster init --name my-cluster --distribution Vanilla \
  --mirror-registry 'docker.io=https://registry-1.docker.io'
````

## Troubleshooting

### 1. Cluster Creation Fails

**Symptom:** `ksail cluster create` fails with Docker errors.

````bash
docker ps      # Verify Docker is running
df -h          # Check disk space
docker system prune -f
ksail cluster create
````

### 2. Nodes Not Ready

**Symptom:** `kubectl get nodes` shows `NotReady` status.

````bash
kubectl describe node <node-name>
kubectl get pods -n kube-system | grep cilium
kubectl rollout restart daemonset/cilium -n kube-system
````

### 3. LoadBalancer Stuck in Pending

**Symptom:** LoadBalancer service never gets EXTERNAL-IP. Ensure `loadBalancer: Enabled` is set in `ksail.yaml`, then run `ksail cluster update`. See [LoadBalancer Configuration](/configuration/loadbalancer/).

### 4. Port Already in Use

**Symptom:** `address already in use` error during cluster creation. List and delete conflicting Kind clusters:

````bash
kind get clusters
kind delete cluster --name <conflicting-cluster>
````

### 5. Out of Memory or Disk Space

**Symptom:** Cluster creation fails with resource errors.

````bash
ksail cluster init --name my-cluster --distribution Vanilla --control-planes 1 --workers 0
# WARNING: removes ALL unused images and volumes — can affect other Docker projects
docker system prune -a --volumes
````

## Advanced Topics

### Multi-Node Clusters

Create production-like topologies:

````bash
ksail cluster init --name multi-node --distribution Vanilla --control-planes 3 --workers 5
````

### Kubernetes Version Selection

Set the node image in `kind.yaml` to pin a Kubernetes version. Check [Kind release notes](https://github.com/kubernetes-sigs/kind/releases) for available images:

````yaml
# kind.yaml
nodes:
  - role: control-plane
    image: kindest/node:v1.31.0
  - role: worker
    image: kindest/node:v1.31.0
````

### Custom CNI

Override the default CNI:

````bash
ksail cluster init --name my-cluster --distribution Vanilla --cni Calico
````

### Port Mappings

Expose services on host ports:

````yaml
# kind.yaml
nodes:
  - role: control-plane
    extraPortMappings:
      - containerPort: 30080  # NodePort service
        hostPort: 8080
        protocol: TCP
````

Access services at `http://localhost:8080`.

### Storage Configuration

Enable persistent storage:

````bash
ksail cluster init --name my-cluster --distribution Vanilla --csi Enabled
````

### Integration with Native Kind

The `kind.yaml` generated by `ksail cluster init` is standard Kind config — use it directly without KSail:

````bash
kind create cluster --config kind.yaml
kind export kubeconfig --name my-cluster
````
