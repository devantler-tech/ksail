Description:
  Update a cluster

Usage:
  testhost update [options]

Options:
  -c, --context <c>                               The kubernetes context to use. [default: kind-ksail-default]
  -k, --kubeconfig <k>                            Path to kubeconfig file. [default: {UserProfile}/.kube/config]
  -t, --timeout <t>                               The time to wait for each kustomization to become ready. [default: 5m]
  -kp, --kustomization-path <kustomization-path>  The path to the root kustomization directory. [default: k8s]
  -dt, --deployment-tool <Flux|Kubectl>           The Deployment tool to use for applying a kustomization. [default: Kubectl]
  -p, --publish                                   Whether to publish manifests on update.
  -v, --validate                                  Validate project files on update.
  -r, --reconcile                                 Reconcile manifests on update.
  -?, -h, --help                                  Show help and usage information
