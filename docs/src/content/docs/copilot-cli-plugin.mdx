The [GitHub Copilot CLI](https://docs.github.com/en/copilot/how-tos/copilot-cli) supports [installable plugins](https://docs.github.com/en/copilot/concepts/agents/copilot-cli/about-cli-plugins) that bundle agents, skills, hooks, and MCP server configurations. KSail ships a first-party plugin that:

- Registers the `ksail` [MCP server](/mcp/) (via `ksail mcp`), giving Copilot direct access to consolidated `cluster_*`, `workload_*`, and `cipher_*` tools.
- Adds a **ksail** skill so Copilot knows when to reach for ksail and links back to this documentation site instead of guessing at flag semantics.

## Install

```bash
copilot plugin marketplace add devantler-tech/ksail
copilot plugin install ksail
```

Verify:

```bash
copilot plugin list
```

Inside an interactive session:

```
/plugin list
/skills list
```

## Prerequisites

- `ksail` available on `PATH` — see the [Installation Guide](/installation/).
- Docker running (required for all local cluster providers).

## What it contains

See the plugin source in the repo: [`copilot-plugin/`](https://github.com/devantler-tech/ksail/tree/main/copilot-plugin).

| File | Purpose |
|------|---------|
| `plugin.json` | Plugin manifest |
| `.mcp.json` | Registers the `ksail` MCP server (`command: ksail`, `args: ["mcp"]`) |
| `skills/ksail/SKILL.md` | ksail expertise skill |

## Versioning

The plugin version tracks ksail releases. Each `v*` tag triggers [`cd.yaml`](https://github.com/devantler-tech/ksail/blob/main/.github/workflows/cd.yaml), which rewrites the `version` field in `plugin.json` and [`.github/plugin/marketplace.json`](https://github.com/devantler-tech/ksail/blob/main/.github/plugin/marketplace.json) and commits the bump back to `main`. `copilot plugin marketplace add devantler-tech/ksail` reads the default branch, so installs always see the latest version.
