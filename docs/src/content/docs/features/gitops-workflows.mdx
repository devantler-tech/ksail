Enable GitOps with [Flux or ArgoCD](/concepts/#gitops) for declarative, Git-driven deployments. KSail automatically installs the GitOps engine, scaffolds FluxInstance or ArgoCD Application CRs, and provides commands to package manifests as OCI artifacts, push to the local registry, and trigger reconciliation.

```bash
ksail cluster init --gitops-engine Flux --local-registry localhost:5050
ksail cluster create
ksail workload push
ksail workload reconcile
```

## OCI Artifact Tag

`spec.workload.tag` sets the OCI artifact tag used by both `workload push` and GitOps reconciliation (Flux OCIRepository / ArgoCD Application). Configure it in `ksail.yaml`:

```yaml
spec:
  workload:
    tag: v1.2.3  # default: dev
```

The tag is resolved differently depending on the context:

- **Push** (`ksail workload push`): CLI `oci://` ref > `spec.workload.tag` > registry-embedded tag > `dev`.
- **Reconciliation** (Flux / ArgoCD): `spec.workload.tag` > registry-embedded tag > `dev`.

See [Declarative Configuration](/configuration/declarative-configuration/) for the full `spec.workload` reference.

### Tag drift auto-repair

`ksail cluster update` detects when the live GitOps sync ref (Flux OCIRepository tag or ArgoCD `targetRevision`) has drifted from the configured `spec.workload.tag` and reconciles it in-place — no manual intervention required. This keeps the cluster's GitOps engine aligned with your declared configuration automatically.

## Commands

| Command                    | Description                            |
| -------------------------- | -------------------------------------- |
| `ksail workload push`      | Package and push manifests to registry |
| `ksail workload reconcile` | Trigger GitOps sync and wait for completion — tracks each Flux Kustomization (with dependency ordering) and each ArgoCD Application individually; for Flux, fails immediately on permanent errors (build failures, health check failures, or upstream dependency failures) rather than waiting for the full timeout |

### Automatic Retry

Both `workload push` and `workload reconcile` automatically retry on transient failures:

- **Up to 3 attempts** with exponential backoff (5–30s between attempts)
- **Retryable conditions**: HTTP 429 (rate-limit), 5xx server errors, TCP resets, and context deadline exceeded (common under CI load)
- A warning is printed on each retry attempt so you can follow progress

`workload push` previously only retried low-level OCI writes; command-level retries now wrap the full build-and-push cycle. `workload reconcile` previously had no retry at all.

For `ksail workload reconcile`, the `--timeout` flag (if set) is a **per-attempt** bound — total runtime can reach up to `maxAttempts × timeout` plus backoff.
